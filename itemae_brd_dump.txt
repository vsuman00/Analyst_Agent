[Heading 1] 1. Executive Summary
[Unknown] The Itemae application is an internal Twitter-clone social networking backend built by CyberAgent (Japan) using Kotlin 1.3 / Spring Boot 2.1 / gRPC / MySQL 5.7 and deployed on Google Kubernetes Engine (GKE). The application was originally authored circa April 2019 and has accumulated significant tec
[List Paragraph] End-of-life framework versions ΓÇö Kotlin 1.3.21, Spring Boot 2.1.4, JVM 8 (all EOL)
[List Paragraph] Deprecated build tooling ΓÇö jcenter() repository (shut down 2022)
[List Paragraph] No REST/GraphQL API ΓÇö only gRPC, limiting client diversity
[List Paragraph] No authentication middleware decoupled layer ΓÇö Firebase auth logic embedded in every handler
[List Paragraph] Absence of observability ΓÇö no distributed tracing, structured logging, or metrics
[List Paragraph] Security gaps ΓÇö plaintext credentials in ConfigMap, synchronous blocking Firebase calls
[List Paragraph] Legacy database ΓÇö MySQL 5.7 (EOL Nov 2023), monolithic schema without soft-delete
[Unknown] This BRD defines the complete set of business and technical requirements to modernize Itemae into a production-grade, cloud-native social platform backend capable of serving 100K+ concurrent users with full observability, security, and operational excellence.
[Heading 1] 2. Business Context & Objectives
[Heading 2] 2.1 Business Problem Statement
[Unknown] The legacy Itemae codebase cannot be safely extended or maintained because:
[List Paragraph] Critical dependency vulnerabilities exist in Kotlin 1.3, Spring Boot 2.1, and gRPC 1.19.
[List Paragraph] jcenter() Maven repository is permanently shut down ΓÇö the project cannot be built without local caches.
[List Paragraph] Firebase Admin SDK 5.9.0 makes blocking HTTP calls inside gRPC threads, causing thread starvation under load.
[List Paragraph] The followers bug (getFollowerUserList calls findFollowingOrFollowedUserList with the same parameter as following ΓÇö a copy-paste defect) affects data correctness.
[List Paragraph] No soft-delete on tweets/users creates compliance risks (GDPR/right-to-erasure).
[List Paragraph] Kubernetes config uses extensions/v1beta1 (removed in Kubernetes 1.22+).
